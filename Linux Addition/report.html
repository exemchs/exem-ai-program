01 / Background

왜 Linux / WSL 가이드가 필요한가

현재 랜딩페이지는 macOS + Cursor 기반 가이드만 제공한다. IT 모니터링 엔지니어 대다수는 Linux 서버 환경에서 일상 업무를 수행하며, 윈도우 데스크톱 사용자도 WSL을 통해 동일한 경험을 얻을 수 있다.

1
김동훈 차장님 피드백

"리눅스에서 사용하는 경우와 커서나 다른 툴이 없을때 윈도우에 WSL을 설치해서 사용하는 경우가 추가됐으면 좋겠습니다. 엔지니어들한테 익숙한 플랫폼이기도 하고 리눅스에서 사용법이 추가되면 자연스럽게 WSL랑 이어진다고 생각합니다."

2
핵심 인사이트

Claude Code는 터미널 네이티브 AI 도구다. GUI IDE(Cursor) 없이도 SSH 터미널에서 바로 사용 가능하므로, 엔지니어 워크플로우와 자연스럽게 맞물린다. 별도 에디터 설치 없이 서버에서 곧바로 AI 코딩 보조를 활용할 수 있다.

현재 랜딩페이지 상태

Prerequisites 섹션에 macOS/Windows 토글이 있으나, Windows는 Cursor + PowerShell 기반. 팝업 공지에 "추후 Linux/WSL 지원 예정" 문구만 포함된 상태. 실제 가이드는 아직 없음.

02 / Linux Setup

Linux에서 Claude Code 설치

네이티브 인스톨러 한 줄이면 끝난다. Node.js도 필요 없다.

System Requirements

ItemRequirement
OSUbuntu 20.04+, Debian 10+, Alpine 3.19+
RAM4GB+
NetworkInternet connection required
ShellBash, Zsh

Node.js는 네이티브 인스톨러 사용 시 불필요. npm 설치 방식은 deprecated. Git은 Linux에서는 필수가 아님 (Windows에서만 Git for Windows 필요).

Installation

bash - Linux Terminal
# 1. Install Claude Code (native installer)
$ curl -fsSL https://claude.ai/install.sh | bash

# 2. PATH setup (if needed)
$ echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc

# 3. Verify installation
$ claude --version
Claude Code v1.0.62

# 4. Run diagnostics
$ claude doctor
All checks passed!

Troubleshooting

SymptomCauseSolution
command not found: claude PATH not set Add ~/.local/bin to PATH
Install process Killed RAM < 4GB sudo fallocate -l 2G /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile
TLS/SSL error Missing certs sudo apt-get install ca-certificates
Search not working Missing ripgrep sudo apt install ripgrep
Corporate proxy HTTPS blocked Set HTTPS_PROXY env variable
Corporate CA cert Custom cert chain NODE_EXTRA_CA_CERTS=/path/to/cert.pem
03 / WSL Setup

Windows WSL에서 Claude Code 설치

WSL2 + Ubuntu 조합이 표준. PowerShell 한 줄로 WSL 설치 후, 동일한 Linux 명령으로 Claude Code를 설치한다.

1

WSL 설치

PowerShell (관리자 모드)에서 실행. WSL2 + Ubuntu가 기본 설치된다. 이후 PC 재시작.

PowerShell (Admin)
PS> wsl --install
Installing: Windows Subsystem for Linux...
Installing: Ubuntu...
The requested operation is successful.

Requirements: Windows 10 build 19041+ or Windows 11 / BIOS virtualization (VT-x/AMD-V) enabled

2

Ubuntu 초기 설정

재시작 후 Ubuntu 터미널이 자동 실행. 사용자명/비밀번호 설정 후 업데이트.

Ubuntu (WSL)
$ sudo apt update && sudo apt upgrade -y
3

Claude Code 설치

Linux와 완전히 동일한 명령어.

Ubuntu (WSL)
$ curl -fsSL https://claude.ai/install.sh | bash
Claude Code installed successfully!

$ claude --version
Claude Code v1.0.62
4

인증 (로그인)

WSL에서 브라우저가 자동으로 열리지 않을 수 있다. 두 가지 방법으로 해결.

Ubuntu (WSL)
# Method A: Set Windows browser path
$ export BROWSER="/mnt/c/Program Files/Google/Chrome/Application/chrome.exe"
$ claude

# Method B: Copy URL manually from terminal
$ claude
Open this URL to authenticate:
https://claude.ai/oauth/authorize?...
Press 'c' to copy URL to clipboard

WSL 핵심 주의사항: 파일시스템 성능

프로젝트 파일은 반드시 Linux 파일시스템(~/projects/)에 저장해야 한다. /mnt/c/(Windows 드라이브)는 약 9배 느리며, 검색 결과 누락, 빌드 속도 저하가 발생한다. WSL2는 bare-metal Linux 대비 약 87% 성능.

VS Code + WSL 연동

Windows에 VS Code 설치 > Remote - WSL 확장 설치 > WSL 터미널에서 code .로 열기. IntelliSense, 디버깅 등 모든 기능이 Linux 환경에서 동작한다.

핵심 메시지: 왜 터미널 AI인가

IT 모니터링 엔지니어의 작업 환경은 SSH 터미널이다. GUI IDE를 설치할 수 없는 운영 서버에서도 Claude Code는 바로 사용 가능하다. "코드를 만들고 끝"이 아니라, "만들고 → 실행 → 검증"까지 한 사이클을 터미널 안에서 완결한다. 이것이 Cursor나 다른 GUI AI 도구와의 결정적 차이점이다.

05 / Questions for Engineers

엔지니어 대상 사전 질문 리스트

프로그램 구성의 정확도를 높이기 위해 확인이 필요한 사항. 김동훈 차장님 외 다른 엔지니어분들에게도 공유 가능.

업무 환경

  • 01 주로 사용하는 Linux 배포판은? (Ubuntu, CentOS, RHEL 등)

  • 03 사내 인증서(CA certificate)를 별도로 사용하는가?
  • 04 개발/운영 서버 평균 RAM은? (4GB 미만 서버 존재 여부)
  • 05 SSH 접속으로 원격 서버에서 작업하는 비율은?

  • 08 가장 많이 작성하는 스크립트 종류는? (bash, python, SQL 등)

06 / Implementation Plan

랜딩페이지 반영 방안

기존 랜딩페이지에 Linux/WSL 콘텐츠를 반영하는 3가지 방안.

A. OS 탭 확장

Prerequisites 섹션의 macOS/Windows 토글에 Linux 탭 추가

  • 기존 구조 유지, 최소 변경
  • 빠른 구현 가능
  • WSL 가이드 넣기에 공간 부족
  • 엔지니어 특화 예시 불가

B. 별도 섹션 추가

Prerequisites 하단에 "Linux / WSL" 독립 섹션

  • 충분한 공간 확보
  • 엔지니어 예시 포함 가능
  • 페이지 길이 증가
  • 비전공자 대상과 혼재

A + C 조합

기존 페이지에 Linux 탭(간단) + 상세 가이드는 서브페이지

  • 기존 페이지 경량 유지
  • 엔지니어 맞춤 콘텐츠 충분
  • 트러블슈팅 + 예시 모두 수용
  • 추가 개발 시간 필요
